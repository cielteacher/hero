### 📊 数据流清晰化

```
Cmd 层决策 → IMU状态 → gimbal_fb (FB_GYRO 或 FB_MECH)
                             ↓
Execute 层直接使用 → 选择 PID 参数 → 应用反馈源
                             ↓
Motor_Control() 统一执行 PID 控制
```

### 💡 设计理念

**分层职责**：
- **Cmd 层**：决策（IMU状态 → 反馈源）
- **Execute 层**：执行（反馈源 → 控制动作）
- **互不重复**：各层只做一次判断

**接口稳定**：
- 只需在 Cmd 层调整反馈源决策逻辑
- Execute 层无需修改，简化维护
